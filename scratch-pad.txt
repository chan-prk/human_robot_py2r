หน้า intro
  บอกว่าแต่ละหน้ามีวัตถุประสงค์อะไร
    หน้าแรกเอาโค้ดเก่ามาทำให้รันได้
    หน้าสอง เพราะว่าถ้าดูโค้ดในส่วนของการ train model เจ้าของข้ามส่วนที่เป็น model/hyperparameter tuning. ก็เลยต้องใส่เองเพื่อความสมบูรณ์ 
            อธิบายด้วยนะว่า เป็นไปได้ว่าระหว่างการแข่งขันเจ้าของอาจจะมีโค้ดส่วนนี้เพื่อหา algorithm หรือ parameter ที่ดีที่สุด แต่หลังจากสิ้นสุดการแข่งแล้ว
           เจ้าของซึ่งรู้คำตอบอยู่แล้วน่าจะ hard code ลงไปใน script ที่แชร์ให้สาธารณะ โดยที่ผู้ที่มาดาวน์โหลด script สามารถ
           นำไปใช้ได้โดยไม่ต้องเสียเวลารัน script เป็นเวลานานเพื่อหาคำตอบว่า algorithm/parameter ไหนดีที่สุด)
    หน้าสามกับหน้าสี่จะเป็นการ เขียนโค้ดขึ้นใหม่โดยอิงจากโค้ดใน section ก่อนหน้า
      ส่วนหลักๆในโค้ด ตามที่เราได้เห็นกันมา ก็เป็นการสร้าง feature และ train model ซึ่งเราจะอธิบายส่วนนี้ในหน้าสาม
            นอกจากนี้ ถ้าสังเกตใน original code ก็จะมีโค้ดส่วนที่เป็น data visualization ที่ถูก comment ไว้ ในหน้าที่สี่เราจะแสดง plot ที่ถูกสร้างใน step นี้


ลิงค์ไปยัง kaggle ใส่ตรง overview ของแต่ละหน้า บอกด้วยว่า เราจะยืนยันว่าโค้ดใช้การได้ด้วยการที่ดูว่าคะแนน เมื่อทำการ submission ใกล้เคียงกับคะแนนของเจ้าของโค้ดใน leaderboard หรือเปล่า
  จะพูดเหมือนเดิมทุกๆหน้ามั้ย ถ้าทำ จะทำอย่างไรให้ดูไม่น่าเกลียด เช่นใส่ใน quote หรืออะไร
  หรือจะพูดยาวๆแค่หน้าแรก แล้วก็ ในหน้าอื่นๆ เราก็จะโชว์แค่ private score และบอกว่ามัน match กับ score ในหน้าก่อน ก็พอ
